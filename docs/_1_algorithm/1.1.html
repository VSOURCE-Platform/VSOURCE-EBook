一、Anaconda环境

为什么要用Anaconda不用原生的自建Python形式？

  1. 环境方便迁移。
  2. 一个应用配一套环境，是一个基本的常识，而往往开发者面对的是多个应用，最佳的做法是对每一个应用配套一个合适的环境。
  3. 各类包的版本指定容易，甚至包括Python自己。一个亲身的案例在于，某一个应用需要Python3.5，而另一个应用需要Python3.8，Anaconda是最好的解决方案。

二、安装Anaconda环境

2.1 方法一：从Anaconda官网下载

步骤一：从官网下载对应系统的产品，https://www.anaconda.com/products/individual 步骤二：将Conda的bin目录加入到环境变量即可使用。 换源与相关问题：http://www.ecohnoch.cn/Ecohnoch-handbook/#/env/1

2.2 方法二：Docker拉取镜像

步骤一：安装Docker，参考下列教程在指定的操作系统上安装：

  1. MacOS Docker安装
  2. Windows Docker安装
  3. CentOS Docker安装
  4. Debian Docker安装

步骤二：Docker镜像加速。

进入相关的镜像网址。

步骤三：搜寻Anaconda相关镜像。

docker search anaconda

步骤四：拉取Anaconda官方镜像，大概1G左右，要下载，速度比较慢，耐心等待一下。

docker pull continuumio/anaconda3

步骤五：运行镜像。

docker run -i -t -p 12345:8888 continuumio/anaconda3 /bin/bash

说明1. -i: 是  以交互模式运行容器，通常与 -t 同时使用；-t: 为容器重新分配一个伪输入终端，通常与 -i 同时使用；-p: 指定端口映射，格式为：主机(宿主)端口:容器端口，端口只需要不和其他的应用重复就可以了。

  1. TensorFlow依赖gpu环境，Docker的容器是否能正常的访问GPU（尤其是在Windows下）还是问题，所以尽量以第一种方式来安装是更好的。

三、Anaconda说明

  1. 创建一个指定Python版本的Anaconda环境。
conda create -n 环境名字 python=3.6
  1. 查看各种built-in lib和第三方库的位置。

四、实践一

  1. 在自己的电脑上安装Anaconda。
  2. 安装Anaconda后，分别创建两个虚拟环境，分别叫my_env1和my_env2，采用的Python版本分别为3.6和3.8。
conda create -n my_env1 python=3.6
conda create -n my_env2 python=3.8
  1. 在my_env1的环境下安装flask_autoapi，并测试运行。
pip install flask-auto-api -i https://pypi.python.org/simple
pip install flask==1.1.2
pip install flask-RESTFUL==0.3.8

python
>>> import flask_autoapi
>>>

切换到my_env2环境，并测试运行是否有flask_autoapi。

conda deactivate
conda activate my_env2
python
>>> import flask_autoapi
Trace back...

并切换回my_env1环境。

  1. 在my_env1的环境下，创建一个空的目录叫做my_service，并在该空目录下创建一个叫做sin.py的文件，sin.py如下：
import math

def naive_sin(x):
    x_float = float(x)
    return str(math.sin(x_float))

在同级目录下创建config.py文件，config.py如下：

import flask_autoapi
service_config = {
    'service_dir'               : './',
    'service_python_filename'   : 'sin',
    'service_python_interface'  : 'naive_sin',

    'service_input_params' : {
        'x': 'input number'
    },

    'service_output_params' : {
        'sin_result': 'sin result'
    },

    'deploy_mode'   : 'restful',
    'deploy_port'   : '12345',
    'service_route' : '/sin'
}

if __name__ == '__main__':
    flask_autoapi.launch(service_config)

然后运行config.py。

  1. 访问服务
curl http://ip:12345/sin?x=6.28

五、实践二

场景：上述作业一的情况下，已经可以本地的访问sin的服务了，如果现在要将这个服务部署在某一个现场，现场的操作人员并不懂得任何Python相关的知识，只是希望能双击点开一个东西，然后就可以正常用这个服务了，如何操作？

  1. 制作工程目录client。
  2. 将anaconda/envs/my_env1的所有文件拷贝到client
  3. 将上述my_service下所有的文件拷贝至client目录下。
  4. 制作双击脚本启动服务app.sh，并修改权限。

app.sh:

nohup ./bin/python -u config.py > app.log 2>&1 &

修改权限：

sudo chmod 777 app.sh
  1. 启动服务。
bash app.sh
  1. 将整个client文件打包，即可部署到任意一个【相同操作系统】但可以没有装任何的环境的机器下运行了。

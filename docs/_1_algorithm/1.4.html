一、图像分类（Image Classification）

也就是输入一张完整的图像，然后判别图像的类别，这里不会考虑到目标物体的位置、占据的比例等，直接就是整张图输入得到类别。

图像分类基本是所有的计算机视觉任务的基础。人脸识别、说话人识别、虹膜识别等都属于图像分类最直接的下游任务；而目标检测、语义分割等都基于分类任务。

从数据的角度出发，图像分类只需要两种数据，一堆图像，和对应的类别标签。例如<image, label>这样，image往往都是一个具体的三维数字矩阵，label往往都是一个整数，直接表示类别。

图像分类的发展往往都是靠标准数据集和网络结构推进的，一般的套路都是总结现有网络结构的问题，然后尝试提出新的网络结构来解决这些问题，并且在标准数据集上可以达到更好的精度。

1.3 图像分类下游任务三：fine-grained图像分类

中文名：细粒度图像分类。

人脸识别实际上也属于一种细粒度的分类，但是人脸识别的实际应用方面是趋向于【比对】的形式。但是诸如鸟类的类别、花草的类别、垃圾的类别、同一个产品不同的产地、同一个产品不同的生产厂家等这样的问题是相比于传统图像分类更难的问题。

1.4 图像分类下游任务四：半监督图像分类

半监督分为两类，一类是self-training，另一类是co-training。

self-training，也就是先在已有的数据上训练一个差不多的模型，然后与预测一批没有标签的图像，把置信度最高的预测标签直接用来作为标签，然后放进原来的带标签数据集中，继续训练，然后继续收集模型打好的置信度高的标签，直到已标注的数据集基本不会发生变化为止。

co-training，和自我学习一样，假设现在有数据集，每组数据有2个特征(feature): x1 和 x2。我们把其中有标签的那部分数据集称为L （表示为[x1,x2,y]），没有标签的数据集称为U (表示为[x1,x2])。

  1. 初始化数据，把L 分为L1 ([x1,y]) 和L2 ([x2,y])。这里注意，L 是一个数据集，里面包涵了很多数据，我就顺便提醒一下。
  2. 重复：a) 分别用L1和L2训练出一个模型 F1和F2，b) 分别用模型F1和F2去预测U (给U打标签)，同样这里也是只选出最有把握的一些结果。 c) 把F1预测的结果放入L2，把F2预测的结果放入L1 (交叉放置)。d) 更新L 和 U
  3. 直到数据集不发生变化 (和自我学习一样)

参考综述：https://zhuanlan.zhihu.com/p/179233410

数据集和Benchmark：https://paperswithcode.com/task/semi-supervised-image-classification

二、目标检测（Object Detection）

这个是相比图像分类，目前深度学习、计算机视觉应用最多的任务，是除人脸识别、OCR以外落地项目最多的任务。这个任务便是输入一张图像，得到图像里某些物体的位置。

从数据的角度出发，目标检测的任务需要的数据，是一堆图像，以及每个图像中对应的希望被检测物体的物体类别、物体位置（用左上角的坐标和右下角的坐标来表示一个框）。有时候你甚至可以完全不知道目标检测的原理，只需要知道喂入网络的数据是什么形式的，而且有哪些网络效果比较好，你就可以直接上应用的战场去做项目产品的落地了。

目前普遍来说有两种方法，分别是one-stage方法和two-stage方法。

one-stage的方法就是一步到位，网络的特征图中直接包含类别标签和位置标签的信息，也就是特征并不仅仅只是类别特征，而是类别+位置两种特征的信息，这样对网络的特征图部分要做很多的工作。其优点是速度非常的快，适用于高并发和多物体的检测。比较有名的网络是SSD、Yolo系列（Yolov3、Yolov4、Yolov5）等。其中Yolo-v5的模型应用于大规模的比赛中，可以作为目标检测的Baseline模型。

two-stage的方法是将网络结构分为边框回归和分类两种问题，第一个阶段主要构造边框回归的结构，意图在找寻到大量的边框，然后第二个阶段对这些边框进行分类，标注置信度和类别，最后进行整合分析和去框，其优点是准确率会较高，但缺点是速度往往较慢，不适用于对检测速度有要求的任务。比较有名的网络是Faster-RCNN、RetinaNet等。

三、语义分割（Semantic Segmentation）

这个是仅次于图像分类、目标检测，计算机视觉应用最多的任务。其任务的含义就是给定一张图像，对图像里的【每个像素】进行语义的多分类。比如你设置一个马路的照片有四个语义，分别是马路、汽车、行人、其他，那么输入这张马路的图像，你希望得到的图像便是只有四个颜色的图像，不同的颜色分别表示四个语义。

从数据的角度出发，语义分割的任务需要的数据，是一堆图像，以及对应图像标注好的一个mask图像，原始的图像是[h, w, channel]的shape的图像，而mask图像是[h, w]的形式，里面的每一个值都是一个类别值，表示不同的语义类别。同样的有时候你甚至可以完全不知道语义分割的原理，只需要知道喂入网络的数据是什么形式的，而且有哪些网络效果比较好，你就可以直接上应用的战场去做项目产品的落地了。

其应用价值主要在智能驾驶、遥感卫星图像方面，比如识别道路信息、统计耕地的面积等。

语义分割的发展还是比较稳健的，按时间顺序有以下的一些网络结构：

  1. EfficientNet-L2+NAS-FPN：目前的benchmark，2020。
  2. CFNet：基于ResNet-101，2019。
  3. DeepLab-V3+：目前用的最多的、效果最稳定的网络，2018。
  4. PSPNet：2016。
  5. FCN：2016。
  6. UNet、SegNet：很早。

实际上网络结构往往目前都采用DeepLab-v3+，稳定且效果好。

数据集和Benchmark：https://paperswithcode.com/sota/semantic-segmentation-on-pascal-voc-2012

四、图像生成（Image Generation）

图像生成任务就是根据一段【噪音】，去生成某一类图像的任务，比如人脸图像生成、卡通人物生成、换装生成、场景图像生成等。

这是一个和分类任务【相反】的任务，在分类任务中，我们的目标是提取到图像关于某个类别的特征，然后根据这些类别的情况进行分类。那么同样，我们也可以直接得到一个特征向量，反向的解码出对应的图像，这个是生成任务。其应用面相对较窄，但是在文娱方面依然有很大的应用潜力。

从数据的角度出发，图像生成的任务需要的数据，仅仅就是一堆图像。然后构建特征提取，以及特征解码的网络的进行训练，在进行生成的时候，仅仅只需要特征解码的网络，以及一段随机生成的特征向量，便能够得到对应的图像。这个任务相比于其他的任务视觉冲击更大，而且操作起来非常的简单，所以经常可以在网上看到类似的东西。

比如：

  1. 每次点进去，会得到一张生成的人脸，这个人脸并不存在世界上，只是计算机根据特征生成的： https://thispersondoesnotexist.com/
  2. 每次点进去给你一张生成的人脸和真实的人脸，哪张人脸是真实的呢？： http://whichfaceisreal.com/
  3. 每次点进去会给你一张二次元的图像，这个图像是唯一的，没有人画过它，可以用来作为头像，而且永远不会撞头像，也不需要考虑版权问题：https://www.thiswaifudoesnotexist.net/

现在常常采用的方法是GANs和VAE的方法。想要达到好的生成效果，图像的量要非常的大、质量要非常的好。高清的人脸图像数据集，一张图像都是非常的巨大的，而且对机器的要求也非常的高。

相关数据集和Benchmark：https://paperswithcode.com/area/computer-vision/image-generation

六、其他可能的各种任务

计算机视觉的各种任务都基于计算机图像，而且往往很多套路都是可以通用的，比如单张图像的效果好了之后，自然会联想到3D的情况、联想到视频的情况、联想到标签不够半监督的情况、联想到其他的视觉领域。而且人脸、视觉场景、道路场景、工厂环境、办公场合等等的视觉场合也都是非常不一样的，都有特定的套路和操作。

所以有时候，基本的问题进行一些组合，就可以提出很多其他的任务，比如OCR就是一种集大成者，堪称是除了人脸识别以外应用最广的一种任务，比目前智能驾驶的业务场景要更多。OCR便是图像中字符的提取，比如你截了一张图，图里存在一些文字，那么可以采用OCR的技术来提取出这样的文字，比如车牌号的识别等等。OCR便是一种视觉任务组合的结果，首先要进行文字区域的检测，检测后要对文字区域进行划分和文字的识别，对于句子型的文字还需要结合语义分析等自然语言处理中的手段。

所以研究视觉问题，要首先明白任务是什么，和哪些上游任务是紧密相关的，而且也要格外关注当前面对的视觉场景，是哪一种，有哪一些特别的套路和操作。另外需要时刻关注https://paperswithcode.com/task/中相关的benchmark和dataset的信息，以上强调过很多次，有时候你需要用什么技术的时候，甚至可以完全不知道里面的原理，但只需要知道数据怎么喂入网络、有哪些网络效果比较好、有哪些数据你可以用这三点，基本就可以以很快的速度将结果复现出来，这是在工业应用中，非常重要的技能。在学术研究的时候，也可以将目前的benchmark复现出来达到对应的结果，然后分析可能存在的问题，结合过去的一些研究，或者自己提出一些创新的手段去改进，然后突破这个benchmark，就是一个新的研究，或者也可以在这个基础上扩宽任务领域，或者提出更多的数据等。

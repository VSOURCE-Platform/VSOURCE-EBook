一、训练环境安装

1.1 训练环境我们需要TensorFlow-gpu环境。

参考：为什么神经网络训练需要GPU环境？

浓缩理由如下：

神经网络计算是高度并行的，GPU适合并行计算，尤其是卷积神经网络。高度并行任务指的是一个任务可以被分割为多组独立的小任务。

如∑i=1ni\sum_{i=1}^{n}{i}∑i=1n​i，直观来看待的话计算复杂度为Θ(n)\Theta (n)Θ(n)，但是如果我们分割为两两一组来计算，我们计算一组的时间单位为1，只需要计算n2\frac{n}{2}2n​组，这样计算复杂度为Θ(n2)\Theta (\frac{n}{2})Θ(2n​)。同理，如果一次计算的时间单位内我们可以并行计算kkk次，那么计算复杂度则为Θ(nk)\Theta (\frac{n}{k})Θ(kn​)，可以大幅度的降低总体时间，但单位时间内的计算量也会大幅度的增加。

1.2 TensorFlow-gpu的安装的一些说明

我们可以看到一些相关的WIKI下，有比较详细的TensorFlow-gpu的安装，例如：TF-Wiki

【注意，但是我们一般不推荐按照这样的Wiki来安装】

理由如下：
（1）wiki中往往介绍的安装的tf-gpu的版本，并不是我们所希求的版本，例如上述的wiki中，安装的是tf-gpu-2.1的版本，而TensorFlow一个很大的毛病在于，每一个tf的版本差异都【相差巨大】，高版本很难兼容低版本，同理低版本也很难具有高版本的特性。
（2）按照各种wiki安装，很容易把系统的环境整坏，导致再也安装不上TensorFlow-gpu，甚至可能要走好几遍Anaconda的安装流程，所以最好的办法，就是自己要理解TensorFlow-gpu按照的各种依赖和套路。

1.3 TensorFlow-gpu的安装流程

安装流程以Windows电脑为例，而Mac、Linux都类似，要注意，安装之前确认目标机器上具备GPU。

（步骤一）安装Tf-gpu的gpu依赖部件。

主要包含Cudatoolkit+cudnn两部分。要注意版本要严格和tf-gpu的版本匹配，就算2.1.1和2.1.0的版本也会出现不适配的问题，所以一定要注意。官方的版本匹配

我们所安装的环境的具体的版本号与安装方式：

安装的组件 具体版本 安装方式
TensorFlow-gpu 2.4.0 pip install tensorflow-gpu==2.4.0
CUDATOOLKIT 11.0 https://developer.nvidia.com/cuda-11.0-download-archive
CUDNN 8.0.5 https://developer.nvidia.com/zh-cn/cudnn

附：

TensorFlow 对 NVIDIA 显卡的支持较为完备。对于 NVIDIA 显卡，要求其 CUDA Compute Capability 须不低于 3.5，可以到NVIDA官网。

查询自己所用显卡的 CUDA Compute Capability。目前，AMD 的显卡也开始对 TensorFlow 提供支持，可访问 这篇博客文章 查看详情。

（步骤二）添加环境变量

（1）TensorFlow-gpu==2.4.0下载完成后，我们暂时不需要做什么。

（2）CUDATOOLKit==11.0下载完成后，我们安装教程进行安装，安装在默认的目录为C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.0，然后我们需要将这个目录下的bin和libnvvp两个文件夹加入环境变量。

（3）将CUDNN下载到的压缩包解压，解压得到的目录下面的文件，拷贝到CUDA的目录，也就是上面那一步骤的v11.0的文件夹下，替换所有的文件即可。

（步骤三）检查版本

（1）先进入对应的conda虚拟环境，然后检查TensorFlow-gpu的版本。

pip list

在下面找到Tensorflow的版本，查看是否为2.4.0。

（2）检查CUDA的版本。

可以看到在上述添加了环境变量的bin目录下，存在nvcc的这个可执行程序，我们直接打开cmd，然后输入nvcc -V，可以查看CUDA是否为11.0版本。

如果nvcc说找不到，说明环境变量没有添加，就把这个目录再添加到环境变量。

（3）检查CUDNN的版本。

在CUDA的目录下，找到include文件夹，下面有一个cudnn_version.h文件，打开后即可看到CUDNN的版本。

二、一个完整的测试

打开一个文本编辑器，新建一个文件命名为test.py，然后保存在某一个位置。

输入：

import tensorflow as tf

if __name__ == '__main__':
	mnist = tf.keras.datasets.mnist
    
	(x_train, y_train), (x_test, y_test) = mnist.load_data()
	x_train, x_test = x_train / 255.0, x_test / 255.0
    
	model = tf.keras.models.Sequential([
		tf.keras.layers.Flatten(input_shape=(28, 28)),
		tf.keras.layers.Dense(128, activation='relu'),
		tf.keras.layers.Dropout(0.2),
		tf.keras.layers.Dense(10, activation='softmax')
	])
 
	model.compile(optimizer='adam', loss='sparse_categorical_crossentropy',
                  metrics=['accuracy'])
    
	model.fit(x_train, y_train, epochs=5)
	model.evaluate(x_test, y_test)

然后，进入我们安装的tf-gpu的虚拟环境，从cmd命令行cd到test.py的目录，然后执行这个python脚本：

python -u test.py

以上的界面，即是在正常的训练一个手写数字识别，可以看到我们确实使用的是gpu的环境（上述的日志中包含了Adding visible gpu devices: 0 ），跑了五轮，准确率在97.54%。关于mnist数据集的细节，以及上述一个脚本的里面的原理，以及pycharm的使用，我们之后会讲，这里还是测试tf-gpu的环境是否安装成功。

三、预测环境安装

  1. 预测环境说明

预测环境我们需要TensorFlow-cpu环境。不需要任何gpu设备，是台15年及以后的电脑就可以了。

具体的操作为，新建一个虚拟环境，pip install tensorflow-cpu==2.4.0即可。非常的简单。

  1. 训练环境和预测环境的区别。

训练我们是在训练我们的神经网络，让其提取到更好的特征，而预测，要注意预测（或者叫推理，英文分别是Prediction和Inference）并不是测试（Testing），预测的目的，是输入一条或多条数据，直接得到这条数据的特征向量，或者直接根据特征向量得到我们想要的结果。

例如对人脸识别来说，训练的目的是输入一个构建好的初始网络，得到一个训练好的网络。测试的目的是输入一个训练好的网络，得到相关的指标。预测的目的是，输入一个或者多个数据，和一个训练好的网络，得到这一个或者多个数据的特征向量，或者直接一步到位得到这些数据的人脸ID。

  1. 为什么预测的时候，就可以不需要gpu的环境了呢？

预测的过程，对网络来说，只是一个前向计算的过程，其计算的消耗，和并行的力度，和训练的时候的反向传播的差别实在是太大。所以预测采用cpu往往就够了，速度有时候甚至比gpu还快。

但是在正规的工业部署中，部署预测或者推理的场景，还是需要采用GPU，并且结合模型压缩、模型加速、模型蒸馏等操作，优化一个训练好的模型，并在GPU上进行预测/推理，可以实现高并发的预测/推理。

这样的好处是速度极快，但同样GPU的价格远远高于CPU，其代价也是难以承担的。

  1. 预测环境的测试。

进入对应的预测环境虚拟环境，然后输入python进入交互式命令行，输入import tensorflow，成功执行就可以了。

3.4 步骤四，获得授权Token

我们拿到上面的应用的API Key，比如叫做xxxkey，然后点击Secret Key显示出来，比如叫做yyykey，然后在浏览器打开下面的链接，要注意把下面的【xxxkey】的区域替换成你刚刚复制的（包括大括号一起替换），然后把【yyykey】也替换成刚刚显示的Secret Key。

https://aip.baidubce.com/oauth/2.0/token?grant_type=client_credentials&client_id=【xxxkey】&client_secret=【yyykey】&

浏览器打开上面的链接后，会返回一堆文本，我们在里面找到【access_token】，将其保留和复制，记录在一个地方。

5.5 步骤五，编写脚本来实现人脸对比。

例如我这里使用两张人脸图像分别叫2.jpg和35.jpg。

然后我们建立一个文件夹，比如叫做test_api，把两张图片放在文件夹下。在这个目录下，我们创建一个test.py的脚本，输入：

import requests
import base64
import json

def base64_encode(image_path):
    with open(image_path, 'rb') as f:
        image_bytes = f.read()
    return base64.b64encode(image_bytes)

if __name__ == '__main__':
    face_image1 = './2.jpg'
    face_image2 = './35.jpg'	
    token = '这里放步骤四拿到的Token'
    api = "https://aip.baidubce.com/rest/2.0/face/v3/match" + "?access_token=" + token
    face_image1_encode = base64_encode(face_image1)
    face_image2_encode = base64_encode(face_image2)
    
    request_params = json.dumps([
        {"image": str(face_image1_encode, "utf-8"), "image_type": 'BASE64', "face_type": "LIVE"},
        {"image": str(face_image2_encode, "utf-8"), "image_type": 'BASE64', "face_type": "IDCARD"}])
    response = requests.post(api, request_params).text
    print(response)
    score = json.loads(response)['result']['score']
    print(score)

不需要TF环境，直接原生的Python环境，运行即可得到输出，可以看到相似度为96，非常的高，可以认为是一个人。

5.6 步骤六，将其做成一个本地的服务。

接上节Python与环境中，在my_env1的环境下，我们制作了一个启动后，即可采用浏览器打开的方式，来访问的服务，我们同样可以将这个百度的API服务，打包成我们自己的一个服务，操作步骤如下。

在上述的文件夹中，新建一个service.py，里面输入：

import requests
import base64
import json
def base64_encode(image_path):
    with open(image_path, 'rb') as f:
        image_bytes = f.read()
    return base64.b64encode(image_bytes)

def face_similarity(face_path1, face_path2):
    face_image1 = face_path1
    face_image2 = face_path2
    token = '这里放步骤四拿到的Token'
    api = "https://aip.baidubce.com/rest/2.0/face/v3/match" + "?access_token=" + token
    face_image1_encode = base64_encode(face_image1)
    face_image2_encode = base64_encode(face_image2)
    request_params = json.dumps([
        {"image": str(face_image1_encode, "utf-8"), "image_type": 'BASE64', "face_type": "LIVE"},
        {"image": str(face_image2_encode, "utf-8"), "image_type": 'BASE64', "face_type": "IDCARD"}
    ])
    response = requests.post(api, request_params).text
    # print(response)
    score = json.loads(response)['result']['score']
    return score

然后制作一个config.py输入：

import flask_autoapi
service_config = {
    'service_dir'               : './',
    'service_python_filename'   : 'service',
    'service_python_interface'  : 'face_similarity',
    'service_input_params' : {
        'face_path1': 'input first face path',
        'face_path2': 'input second face path',},
    'service_output_params' : {
        'similarity_result': 'similarity result'},
    'deploy_mode'   : 'restful',
    'deploy_port'   : '12345',
    'service_route' : '/face_similarity'
}
if __name__ == '__main__':
    flask_autoapi.launch(service_config)

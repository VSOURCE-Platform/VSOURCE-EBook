四、算法架构设计初识

下面是一个深度学习算法的最简单的一个架构，我们自底向上的学习，现在大家有了解过Dataset-Zoo， Model-Zoo的含义。同样的，在更上层，实际上深度学习还有很多的任务，大家目前知道的比较多的是图像分类。

从【学习】的角度而言，我们的做法往往是自底向上，而如果一上来各个层次的都去了解一些，往往也很难学的很深入，容易过一段时间就忘记。而从【工程/业务/应用】的角度而言，我们的思考模型往往又是自顶向下，先从实际提供给客户/用户/服务端的交互接口出发，往下进行分工和包装和设计。

以人脸识别为例，我们自底向上进行思考：

  1. 首先自然是要寻求公开的人脸识别数据集，或者公司内部的数据集；
  2. 然后进一步采用这些数据集，进行模型的搭建和设计；
  3. 再进一步搭建完的模型，根据相关的任务去训练和测试；
  4. 在进一步，对训练完的模型，要进行很完善的测试/验证/各类指标的考核；
  5. 最后一步，将通过测试的模型，进行优化和部署，部署成一个Service对外进行暴露，后续的过程可能涉及产品后续的了。

我们在学习的过程中，就需要时刻的问自己，现在在哪一步，比如现在模型已经搭完了，也简单的用测试集测试完了，那么接下来呢？这个时候会发现实际上处于第3层，这个时候我们要往第4层进行进发，也就是进行完备的评测，可以问一下当前的测试是否可靠。只测试一个指标自然是不可以的，一般的做法要在各个验证/测试集上测试多个指标，然后采集一些实际的数据来测试，随机迭代一些互联网数据来测试。这些测试也仅仅只是对模型效果的测试，在第5层的服务部署成功后，也需要进行更复杂的测试，这里不再阐述。

在上述最简单的一个架构的指导下，我们自然可以很容易的做出一个较为通用的项目工程目录结构：

----Root_dir
		|----Data
    |----Model
    |----Tasks
    |----Evaluation
    |----Deploy

当然，如果你只是一个人的话，上述的五个目录结构，即五个层次，是不可以【并行】去编写的，要先把Dataset相关的接口缕清楚，然后再去做Model，缕清Model的接口，并且实现之后，再去做Tasks....，以此类推。

而如果你是一个团队的话，对上述的五个层次，要先进行【协议的规定】，假设这个团队分为五个小组，分别去解决上述的问题，我们要先约定，Data小组和Model小组之间协作的协议，也就是Data小组会提供哪些接口，然后Model小组先拿到这些接口，不用管具体有没有实现，然后先去设计模型，如此这般，一共需要若干个协议，然后并行的推进即可。

单人进行开发的话，也可以以协议式去并行进行推进，但是我认为没有太大的意义，反而很容易搞出很多的问题，稳妥一些比较重要吧。

现在，我们来定义其中的一个协议：

（协议1）Data和Model的协议。

Data小组，直接在Data目录提供一个Python包。包的导入的方式直接为import Data，包含的接口为如下三个：（1）(x_train, y_train), (x_test, y_test) = Data.load_data(dataset_name='')， （2）meta_data = Data.load_metadata(dataset_name="")， （3）data_index = Data.load_data_index(dataset_name="") 那么，在Data目录下，创建一个__init__.py文件，输入：

# In $(Root_dir)/Data/__init__.py

def load_data(dataset_name):
    pass

def load_metadata(dataset_name):
    pass

def load_data_index(dataset_name):
    pass

同样，协议是双向的，在Model中，也要定义一个包，来接收和调用Data的接口，这里要注意Model中的其他模块最好不要直接import Data。比如在Model中，我们定义一个integration_data.py的文件，输入：

# In $(Root_dir)/Model/integration_data.py

try:
	import Data
    from Data import DataModuleError
except:
    raise DataModuleImportError
    
def load_data(dataset_name):
    try:
    	return Data.load_data(dataset_name)
    except:
        raise DataModuleError

def load_metadata(dataset_name):
    try:
        return Data.load_data(dataset_name)
    except:
        raise DataModuleError

def load_data_index(dataset_name):
    try:
        return Data.load_data(dataset_name)
    except:
        raise DataModuleError
